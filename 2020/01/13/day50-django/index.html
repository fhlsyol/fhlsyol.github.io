前言

有本很有名的Django教学书《Tango with Django》，这名字太棒了，探戈是一种情意浓浓的充满浪漫的双人舞蹈，在加上tango与Django的发音如此的般配，让你在享受与Django共舞的过程中，掌握了一个牛逼的Web应用框架，简直太妙了。

本人绞尽脑汁，最后来了个Django嘿喂狗，正如其名，语文水平喂了狗了，哈哈哈。

由于在写总结博客的时候发现，目前学习的Django知识点零零散散地分散在多个工程中，如果在一篇博客中引入多个工程的冗长的代码 ，不利于对知识点的总结与梳理，所以摒弃了上一篇博客。

这篇博客我打算做一个Django的step-by-step文档，从头开始新建一个工程，一步一步尽量贯穿知识点，会一步步慢慢更新的。

此文档在某些多样性的地方会按照个人的习惯而选择其一 (如创建Django项目的方式、redis缓存的使用方法等)，但一般会在多样性的地方加上外链，以方便查看其他方法。

3. 准备工作

3.1 认识应用

首先认识自己应用文件夹中的文件：

migrations包是模型迁移的包(描述可能不准确，待更新个人知识库)

admin.py可以用来将自己的模型注册到Django的后台管理工具中

models.py可以用来创建数据模型

tests.py可以用来创建测试

views.py可以用来创建视图函数

3.2 配置数据库

我用的mysqlclient

settings.py

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': '你的数据库名',
        'HOST': '你的数据库地址',
        'PORT': 3306,
        'USER': '你的数据库用户(不建议用root)',
        'PASSWORD': '用户密码',
        'CHARSET': 'utf8',
        'TIME_ZONE': 'Asia/Shanghai',
    }
}

这样可以把Django的默认数据库配置成自己的数据库

PS：

create database 新数据库名;
create user 用户名@登录地址;
grant 权限类型 on 数据库 to 用户名;  -- ALL PRIVILEGES是所有权限

3.3 配置静态目录

在工程目录下创建一个名为static的文件夹来存放静态文件

顺手创建css、js和imgs的文件夹用来以后存放相应的静态文件

接着顺手在每个文件夹中创建一个名为.gitkeep的文件，用来保证版本控制的时候将空文件夹也进行同步

然后在settings.py中配置静态文件路径：

STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static'), ]
STATIC_URL = '/static/'

这样在工程中需要静态资源的时候直接写地址头/static/资源地址即可，不用写相对路径了

4. 开始自己的应用

按照上面的步骤做好准备工作后，就可以开始写自己的应用了

4.1 数据模型的生成

ORM(Object Relational Mapping)是通过使用描述对象和数据库之间映射的元数据，将面向对象语言程序中的对象自动持久化到关系数据库中。

4.1.1 ROM正向工程

正向工程在这里就是通过模型类来创建表，直接上例子：

class Subject(models.Model):
    """学科（模型类）"""

    no = models.AutoField(primary_key=True, verbose_name='编号')
    name = models.CharField(max_length=20, verbose_name='名称')

    def __str__(self):
        return f'{self.no}: {self.name}'

    class Meta:
        db_table = 'tb_subject2'
        verbose_name = '学科'
        verbose_name_plural = '学科'


class Teacher(models.Model):
    """老师（模型类）"""

    no = models.IntegerField(primary_key=True, verbose_name='工号')
    name = models.CharField(max_length=20, verbose_name='姓名')
    photo = models.ImageField(upload_to='images', default='', max_length=512, verbose_name='头像')
    subject = models.ForeignKey(to=Subject, on_delete=models.PROTECT, db_column='sno', verbose_name='所属学科')

    def __str__(self):
        return f'{self.no}: {self.name}'

    class Meta:
        db_table = 'tb_teacher2'
        verbose_name = '老师'
        verbose_name_plural = '老师'

上面的代码创建了两个模型类，分别是学科的模型类和老师的模型类，注释说明见表：

代码 说明
models.Model 从django.db中导入
构建模型类时需要继承它
no、name、photo等 属性，对应数据库中的一个字段
AutoField 自增长属性，相当于数据库的auto_increment字段
CharField 字符串
IntegerField 整数
ImageField 图片
ForeignKey 外键
primary_key=True 在数据库中该字段为主键
verbose_name 该字段的备注名
to=Subject 与学科对象外键关联
__str__ 自定义该类的显示
Meta 元对象，用于定义模型类的行为特性
db_table 这个模型类对应的数据库中的表
verbose_name和
verbose_name_plural		 该模型的单数备注名和复数备注名

4.1.2 ROM反向工程

有时，在我们创建工程之前，数据库的数据就已经有了，这时候就要用数据库的数据进行反向迁移，为我们生成模型类，这就是ROM反向工程

由于刚才实现了正向的迁移，所以现在需要把models.py清空，并且把刚才生成的迁移文件0001_initial.py删除，这样才能更好更清晰地看到反向迁移的效果

由于刚才用正向工程在数据库中生成了tb_subject2和tb_teacher2两张表，接下来的反向工程直接用这两张表来生成模型与刚才的模型对比就能更清楚的看出反向工程的特点

确认清除完毕后，Alt + F12 打开终端，执行指令

python manage.py inspectdb tb_subject2 tb_teacher2 > test_app/models.py

这指令将数据库中的tb_subject2和tb_teacher2两张表反向生成模型类的代码，并且重定向到models.py中

注意：

  • 如果不写tb_subject2和tb_teacher2的表名，会默认选中数据库中的所有表

  • 这里并不能使用独立运行的manage.py任务来执行这条指令，因为manage.py不认识重定向的符号，若不重定向，则在控制台打印出反向迁移的代码 (我感觉manage.py里应该有直接重定向到models.py的指令，这里先留个pass占位，以后查到了更新)

成功后：

# This is an auto-generated Django model module.

#   * Rearrange models' order
#   * Make sure each model has one field with primary_key=True
#   * Make sure each ForeignKey has `on_delete` set to the desired behavior.
#   * Remove `managed = False` lines if you wish to allow Django to create, modify, and delete the table

from django.db import models


class TbSubject2(models.Model):
    no = models.AutoField(primary_key=True)
    name = models.CharField(max_length=20)

    class Meta:
        managed = False
        db_table = 'tb_subject2'


class TbTeacher2(models.Model):
    no = models.IntegerField(primary_key=True)
    name = models.CharField(max_length=20)
    photo = models.CharField(max_length=512)
    sno = models.ForeignKey(TbSubject2, models.DO_NOTHING, db_column='sno')

    class Meta:
        managed = False
        db_table = 'tb_teacher2'

我们看到，ROM反向工程自动帮我们把数据库的数据转化为了我们需要的模型类，但是仔细观察一下，一些细节的地方并不能处理的尽善尽美，我们只需要手动微调一下即可。
